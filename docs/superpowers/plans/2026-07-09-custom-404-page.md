# Custom 404 Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace Next.js's default "404 This page could not be found" screen with a branded SISTRO not-found page, using Indonesian copy consistent with the rest of the app.

**Architecture:** Next.js App Router auto-renders `src/app/not-found.tsx` for any unmatched route or an explicit `notFound()` call. It's a single server component — no client state, no data fetching. Visual language is copied from the existing `src/app/error.tsx` and `src/app/maintenance/page.tsx` pages (same dark-navy card-on-glow layout), swapped to the app's actual `brand-*` color tokens (`--color-brand-500` = `#465fff`, defined in `src/app/globals.css:32`) instead of the hardcoded `blue-600` those two files use.

**Tech Stack:** Next.js 16 App Router, Tailwind CSS v4 (CSS-variable theme in `globals.css`), `next/link`, inline SVG icon (no icon package dependency needed).
